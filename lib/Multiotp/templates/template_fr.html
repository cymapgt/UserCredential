PERSONNEL ET CONFIDENTIEL / {MultiotpUserDescriptionUC}

Afin de vous connecter à votre réseau d'entreprise, vous devez utiliser un mot de passe avec authentification forte, également appelé code OTP. Ce code va être généré par votre smartphone et change à chaque utilisation.

Le code généré sera valide pour le compte suivant: {MultiotpUserAccount}
et vous aurez à ajouter votre mot de passe réseau avant chaque code affiché.
et vous aurez à ajouter le PIN suivant avant chaque code affiché: {MultiotpUserPin}
Et sur votre générateur de code, vous devrez taper le PIN suivant: {MultiotpUserPin}

Numéro de série du générateur de code physique associé (token): {MultiotpUserTokenSerial}

Afin d'installer le générateur de code sur votre smartphone, veuillez suivre les trois prochaines étapes.

 

1) Installation d'une application de lecture de QRcode

Si votre smartphone n'a pas encore d'application permettant de lire un QRcode, il vous faut installer une application adéquate (par exemple "ZXing barcode scanner" pour Android, "Quick Scan" pour iOS ou encore "i-nigma" pour Windows Phone).

 

2) Installation de l'application de génération des codes OTP à usage unique

Il faut maintenant installer une application compatible sur votre smartphone. L'installation se fait très simplement en scannant (ou en cliquant) sur le QRcode ci-dessous (valable pour Android, iOS, Windows Phone, BlackBerry):

{MultiotpQrCodeUrl url=http://go.6co.ch/otpauth width=200 height=200}
Google Authenticator / Authenticator / 2 Steps Authenticator
{MultiotpQrCodeUrl url=http://go.6co.ch/motp width=200 height=200}
DroidOTP / iOTP / motp7 / Mobile-OTP Token (Java)

 

3) Configuration de votre générateur personnel

La dernière étape consiste à insérer la clé secrète de génération dans l'application. Veuillez simplement scanner le QRcode imprimer ci-dessous (ou cliquer sur l'image depuis votre smartphone).

{MultiotpQrCodeUserToken width=200 height=200}
{MultiotpUserDescription}

Pour configurer votre générateur personnel sur une autre application ne supportant pas de QRcode, voici les détails de configuration à fournir:

  • Algorithme: {MultiotpUserAlgorithm}
  • Clé secrète (seed): {MultiotpUserTokenSeed}
       (ou {MultiotpUserTokenSeedBase32} en Base32, comme pour Google Authenticator)
  • Nombre de digits: {MultiotpUserTokenNumberOfDigits}
  • Intervale de temps: {MultiotpUserTokenTimeInterval}
  • Prochain événement: {MultiotpUserTokenNextEvent}
